# Docketra Landing Page Positioning (April 2026)

## Purpose
This document defines the current public landing-page messaging for Docketra so product, design, and engineering stay aligned on positioning and trust claims.

## Target audience
Docketra is currently positioned for:
- CA firms
- CS firms
- Law firms
- Compliance and advisory teams
- Operations-heavy service businesses

## Core value proposition
Docketra is a connected operating system for:
1. Getting clients
2. Managing clients
3. Executing work

In practical terms, Docketra combines three modules in one operating flow:
- **CMS** for demand capture
- **CRM** for relationship and pipeline management
- **Tasks** for execution and delivery

## Core operating flow
`CMS -> CRM -> Tasks`

### CMS
- Landing pages
- Public forms
- Embedded forms on existing websites
- API intake for integrations
- Structured lead capture

### CRM
- Lead pipeline
- Owner assignment
- Follow-up tracking
- Notes/activity context
- Conversion tracking

### Tasks
- Dockets
- Internal tasks
- Workbaskets
- QC flow and routing
- Audit history and execution visibility

## Messaging guardrails
- Keep language clear and non-jargony.
- Avoid fake enterprise or billing claims.
- Tie each feature area to an operational outcome.
- Use conversion copy that is confident but not hypey.

## Current pricing status (must remain accurate)
- Early access is currently free.
- Billing/subscriptions are not live yet.
- Copy should explicitly state testing phase and invite feedback.
